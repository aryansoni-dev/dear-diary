# DearDiary Preview Tester Guide

## Preview notice

This is a private preview version of DearDiary. Features may change and occasional bugs may occur. Preview data may be reset before production launch; we will communicate any planned reset. Export anything you want to keep.

Please do not publicly share the APK link or send private journal text in a bug report.

## Installation

1. Open the private APK link on your Android device.
2. Allow installation from the browser or file manager if Android asks.
3. Install and open DearDiary.
4. Confirm the app is named DearDiary and Profile shows `Preview • v1.0.0`.

## What to test

- Onboarding and account creation
- Mood logging and journaling
- History, Calendar, and Insights
- AI Chat and entry reflections
- App Lock and notifications
- Offline writing and reconnection
- Journal export
- Sign-out, sign-in, and account deletion

Use the app naturally, but avoid entering information you would not be comfortable placing in an early preview service.

## Feedback questions

1. Was onboarding understandable?
2. Was creating the first entry easy?
3. Did mood logging feel natural?
4. Was anything confusing?
5. Did the AI responses feel useful?
6. Were Insights understandable?
7. Did the app ever feel slow?
8. Did any feature fail?
9. Did any data disappear?
10. Would you use DearDiary regularly?
11. Which features would you consider paying for later?

## Bug-report format

```text
Device:
Android version:
DearDiary version:
Screen:
What I did:
What I expected:
What happened:
Screenshot/video:
Can it be reproduced:
```

Redact journal content, email addresses, notification text, and other personal details from screenshots or recordings.

## If something goes wrong

- Do not repeatedly retry account deletion or AI actions.
- Keep the app installed if data appears missing, and report the issue first.
- Export your journal before destructive testing when possible.
- Never send passwords, verification codes, tokens, or PINs to the developer.
